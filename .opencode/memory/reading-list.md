# Project Memory: Reading References

Canonical source: `books.md`.

Use this memory when adding or checking `Recommended Reading` sections, connecting a guide page to its source literature, or finding book reviews to link from content.

## Architecture Fundamentals, Qualities, And Styles

- *Fundamentals of Software Architecture* supports architecture fundamentals, characteristics, styles, risk, modeling, career, classification, and role overview pages.
- *Software Architecture in Practice* supports architecture characteristics, documentation overview, architecture evaluation, governance, economic analysis, and requirements framing.
- *Software Architecture Patterns* supports style pages including layered monolith, microkernel, event-driven, microservices, and space-based architecture.
- *Head First Software Architecture* supports component thinking and architecture style pages with a practical design-around-change framing.
- *Software Architecture Metrics* supports architecture metrics and measuring system qualities.
- *Software Architecture: The Hard Parts* supports boundaries, coupling, contracts, data ownership, distributed workflows, and trade-off analysis.
- *Balancing Coupling in Software Design* supports coupling, modularity, and boundary pages.

## Architecture Documentation And Modeling

- *Software Systems Architecture: Working with Stakeholders Using Viewpoints and Perspectives* supports architectural descriptions, producing architectural models, visualizing architecture, architecture evaluation, role overview, and classification.
- Simon Brown's *The C4 Model for Visualising Software Architecture* supports the C4 model page.

## Domain-Driven Design And Events

- *Learning Domain-Driven Design* supports business domain, bounded context, context integration, domain modeling, domain evolution, discovery, and applying DDD in practice.
- *Patterns, Principles, and Practices of Domain-Driven Design* supports DDD foundations, bounded contexts, building blocks, integration, and best practices.
- *Introducing EventStorming* supports discovering domains and contexts.
- *Building Event-Driven Microservices*, *Designing Event-Driven Systems*, and *Enterprise Integration Patterns* support event-driven architecture.

## Data And Distributed Systems

- *Designing Data-Intensive Applications* supports managing operational data.
- *Data Mesh* supports managing analytical data.
- *Building Microservices* and *Microservice Architecture* support microservices-based architecture.

## Requirements

- *Mastering the Requirements Process* supports requirements pages: requirement definition, collection, business use cases, scenarios, selection, writing, functional and non-functional requirements, testing, and communication.

## Communication

- *Communication Patterns: A Guide for Developers and Architects* supports architecture documentation management plus communication pages for visual, written, spoken, effective, remote, and documentation communication.
- *The Culture Map* supports multi-cultural communication.

## Organization, Product, And Leadership

- *Team Topologies* supports engineering organization.
- *Team of Teams* supports alignment.
- *The Culture Code* supports company and organizational culture.
- *The Wisdom of Teams* and *Essential Kanban Condensed* support team effectiveness.
- *The Lean Product Playbook* supports product development.
- *Measure What Matters* supports goals and OKRs.
- *The Personal MBA* supports company and business fundamentals.
- *The Software Architect Elevator* supports architecture role classification.

## Book Review Links

Several references have Substack book reviews. Before adding a new external recommendation, check `books.md` for an existing review link and reuse it when relevant.
